# YouTube 新着要約通知エージェント — 初期要件

毎晩22時に、登録（監視対象）YouTubeチャンネルの新着動画を検出し、要約してメールで通知するエージェント。
本ドキュメントは設計の意思決定ログ（dig セッションの結論）をまとめたもの。

## 全体アーキテクチャ

- **成果物は決定的なスクリプト**（Claude Code は開発ツールとして使う）。毎回 LLM に段取りを考えさせるエージェント型ではなく、手順固定のパイプライン。
- パイプライン: **RSS で新着検出 → Gemini に YouTube URL を渡して要約 → 1通の HTML メール送信**。
- 実行基盤は **GitHub Actions の scheduled workflow**。毎晩 **22:00 JST**（cron `0 13 * * *`、UTC 13:00）。多少の遅延は許容。

## 新着の取得

- 登録一覧の自動追従はせず、**監視チャンネルを手動リスト管理**。
- 各チャンネルの **無認証 RSS**（`https://www.youtube.com/feeds/videos.xml?channel_id=…`）で新着取得。API キー・OAuth 不要。
- チャンネル追加は **ヘルパースクリプトに @handle / URL を渡すと一度だけ channel_id に解決して設定ファイルへ追記**。実行時は解決済み id のみ使用。

## 新着判定

- **既読 ID 記録方式**。`seen.json` に通知済み動画 ID を保存し、RSS にあってリストに無いものだけを新着とする。
- 状態は **リポジトリにコミットして書き戻す**。取りこぼし・二重通知なし。

## 要約

- **Gemini API に YouTube URL を直接渡して要約生成**（字幕スクレイピング不要、Claude API は不使用）。鍵は Gemini API キー1つ。
- 粒度は **詳しめ（小見出し付きの構造化要約・"観なくても分かる"レベル）**、**日本語**。
- 要約不能な動画（メンバー限定 / 非公開 / 年齢制限 / 長すぎ / API エラー）は、**タイトル・チャンネル名・リンク＋「⚠️ 要約できませんでした」を明記し、RSS の概要欄を代替表示**（取りこぼしゼロ）。
- 長尺動画は精度低下・処理増を許容。

## 通知

- **Gmail SMTP（アプリパスワード）** で **`you@example.com`** から同アドレスへ **HTML メール**を送信。
- 各動画に **タイトル・チャンネル名・公開時刻・YouTube リンク**を付与。
- **新着0本の日は送信しない**。

## 必要な Secrets（GitHub Actions）

- Gemini API キー
- Gmail アドレス
- Gmail アプリパスワード

## 実装時に決める細部（着手時に確定）

- Gemini の具体モデル選定（最新の要約向けモデル）と要約プロンプトの文面
- リポジトリ構成・言語（Python 想定）
- Gmail の1日あたり送信制限や Gemini 無料枠の運用上限に触れた場合の扱い
